# 투자 블로그 — 주간 통합 리포트 (spec)

## 1. 목표
매주(주말) 한국은행 보도자료·거시지표·차트추세·내 ETF 포트폴리오를 분석해
**하나의 주간 통합 리포트**로 묶어 블로그에 발행한다.
최종 관점은 **안전마진 기반 가치투자자** 시각의 대응·예측.

## 2. 아키텍처 — 4개 서브에이전트 → 통합 → 발행
```
① kor-bank-analysis  ┐
② macro-analysis     ├─ (통합 정보) ─→ ④ etf-analysis (평가·예측) ─→ 주간 글 조립 → 빌드 → 발행
③ trend-analysis     ┘
```
- ①②③은 병렬 실행 가능. ④는 ①②③ 결과를 입력으로 받는다(depends_on).
- 각 서브에이전트는 `agents/*.md` 지침 파일만 따르고, 자기 섹션 마크다운을 `content/reports/_parts/{발행일}/`에 출력.

## 3. 폴더 구조
```
my-blog/
├── agents/                     서브에이전트 지침 (완료)
│   ├── kor-bank-analysis.md
│   ├── macro-analysis.md
│   ├── trend-analysis.md
│   └── etf-analysis.md
├── content/
│   └── reports/
│       ├── _parts/2026-07-11/  그 주 4개 섹션 초안 (kor-bank.md, macro.md, trend.md, etf.md)
│       │   └── charts/         trend 차트 PNG
│       └── 2026-07-11.md       조립된 주간 통합 리포트(발행 글)
├── src/
│   ├── styles/style.css        디자인(금융/리포트 톤) + CSS 변수
│   └── templates/              글/목록 HTML 템플릿
├── scripts/build.js            마크다운→HTML 정적 빌드 (프레임워크 없음)
├── dist/                       빌드 결과(= GitHub Pages 배포 대상)
├── .github/workflows/pages.yml Pages 자동 배포
├── package.json
└── CLAUDE.md                   작업 규칙(재작성)
```

## 4. 데이터 소스 · 도구 매핑
| 섹션 | 도구 | 이 세션 가용 |
|---|---|---|
| ① 한국은행 | **WebFetch/fetch** (bok.or.kr 보도자료, 무인 자동화용) | ✅ |
| ② 거시 | WebSearch (지표 수집) | ✅ |
| ③ 차트 | Yahoo Finance MCP + KIS MCP(폴백) + matplotlib 3.11(PNG, 한글폰트 AppleGothic) | ✅ 설치됨 |
| ④ ETF | **로컬 파일 `portfolio/portfolio.md`**(구글드라이브 대체) + Yahoo Finance + EDGAR | ✅ (FactSet 미가용→Yahoo·EDGAR 대체) |
| 캘린더 | ❌ 미연결 → 등록 안 함 (거시 STEP9는 블로그 '발표일정' 섹션으로만) |

## 5. 주간 워크플로우 (완전 자동, 스케줄 실행)
매주 주말 예약 작업이 자동으로 아래를 수행한다(사용자 개입 없음).
1. ①②③ 서브에이전트 병렬 실행 → 각 섹션 마크다운 + 차트 PNG 생성.
2. ④ 서브에이전트가 ①②③을 입력받아 포트폴리오 평가·예측 생성.
3. 4개 섹션을 통합 글(`content/reports/{발행일}.md`)로 조립.
4. **개인정보 자동 검사(게이트)**: 통합 글에 절대금액(원/₩ 금액 패턴)이 없는지 스캔. 발견 시 발행 중단하고 알림 — 비중(%)만 허용.
5. `npm run build` → `dist/` 생성.
6. 검사 통과 시 자동 커밋 + 푸시 → GitHub Pages 배포.
7. 실행 결과를 사용자에게 알림(성공/실패, 발행 URL).

## 6. 통합 리포트(발행 글) 목차
1. 이번 주 요약 (핵심 3~5줄 + 경기국면 한 줄 선언)
2. 한국은행 보도자료 분석 (①)
3. 거시지표 분석 + 이번 주 발표일정 (②)
4. 차트·추세 분석 (③, 주봉 차트 이미지 포함)
5. 내 ETF 평가 & 대응 (④, 안전마진/가치투자 관점 — **비율만**, 금액 비공개)
6. 종합 코멘트 & 다음 주 관전 포인트
7. 면책 문구

## 7. 개인정보 · 면책 정책
- **금액 비공개**: 입금·평가 절대금액은 발행 글에 노출하지 않음. **자산 비중(%)·수익률(%)·평가(정성)** 만 공개.
- Google Drive 원본은 로컬 분석에만 사용, 원자료를 저장소에 커밋하지 않음 (.gitignore).
- 모든 글 하단에 면책: "본 글은 정보 제공·개인 기록 목적이며 투자 자문이 아닙니다. 투자 판단과 책임은 본인에게 있습니다."

## 8. 디자인 (금융/리포트 톤)
- 밝은 배경, 높은 가독성. 표·수치·차트 중심.
- 색 체계(CSS 변수): 배경 밝음 / 본문 짙은 회색 / 포인트 딥블루(신뢰) / 강세=초록·약세=빨강·중립=노랑(강도 이모지 🔴🟠🟡🟢💚 유지) / 링크 포인트색.
- **라이트(일반) 모드 고정** — 다크모드 사용 안 함(시스템이 다크여도 라이트 유지). 모바일 가독(표는 가로 스크롤).

## 9. 발행 주기 · 표기
- **주말(토/일)**, 직전 주 데이터 기준. 파일명(내부): 발행일 `YYYY-MM-DD`.
- **표시는 주차 기준**: ISO 8601 주차로 "YYYY년 N주차"(예: 2026년 28주차). 목록·글 상단에 주차를 앞세우고 날짜를 병기.

## 10. 빌드/배포
- `scripts/build.js`: `content/**/*.md`(frontmatter: title/date/tags/description) → `dist/`. 마크다운 라이브러리만 사용, 프레임워크 없음.
- `.github/workflows/pages.yml`: main 푸시 시 빌드→Pages 배포.

## 11. 미해결 / 가정
- (가정) ① 한국은행 출력 형식: "우선순위 순 보도자료 표(제목·날짜·핵심수치·시사점) + 요약 문단". 원문 뒷부분 받으면 교체.
- (가정) ④ 5단계 종합예측 출력 형식은 지침의 서술 방식으로 구성. 원문 확장분 있으면 반영.

## 12. 구현 단계 (승인됨)
- Build-1: 블로그 뼈대(빌드 스크립트·템플릿·디자인·홈·CLAUDE.md·Pages 워크플로우 + 개인정보 검사 스크립트) 구축 + 빈 샘플 리포트로 빌드 검증.
- Build-2: 4개 서브에이전트 실행 → 이번 주 실제 리포트 1건 생성 → 조립 → 빌드(파이프라인 실검증).
- Review: 별도 서브에이전트로 검증(동작·코드·개인정보 노출 여부) → review.md. 문제 수정.
- Deploy: 커밋 + Pages 배포.
- Automate: 검증된 파이프라인을 매주 주말 자동 실행하는 예약 작업 등록(자체 완결 프롬프트, 개인정보 게이트 포함).
  - 로컬 예약 작업은 앱이 열려 있을 때만 실행(닫혀 있으면 다음 실행 시 수행). 완전 무인 실행이 필요하면 클라우드 루틴 등록을 대안으로 안내.
