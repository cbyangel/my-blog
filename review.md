## 4. 권고 (자동화 개선 포함)

**콘텐츠 측면**
- 다음 호부터 시세는 "단일 캡처 시각"을 헤더에 못박고 전 섹션이 그 값을 재사용하도록 하면 편차가 사라진다.
- 무역/국제수지처럼 유사 명칭 지표는 (통관/국제수지) 기준을 괄호로 병기.

**자동화 파이프라인 리스크 (무인 실행 관점)**
- **재현성 낮음.** 이번 호는 (a) 한국은행 bok.or.kr 직접 WebFetch 실패 → WebSearch 언론 교차확인 대체(그래서 nttId "수집 불가"), (b) 시세용 Yahoo MCP/KIS/fetch 실패 → `curl + 샌드박스 해제` 수동 우회로 완성됐다. 두 경로 모두 사람이 개입해야 성립하므로, 스케줄 무인 실행 시 동일 실패가 재현될 가능성이 높다.
- **개선안**
  1. **시세 소스 이중화 + 재시도/백오프.** 1차 Yahoo MCP → 실패 시 KIS(`Kis-mcp`) → 실패 시 stooq/공개 CSV 순으로 폴백. 각 단계 지수 백오프 3회. 이번 세션에서 `yahoo-finance`·`Kis-mcp` MCP는 사용 가능 상태이므로, 샌드박스 해제 `curl` 대신 MCP 우선 사용을 시도하도록 파이프라인 지시를 정비.
  2. **한국은행 수집 안정화.** 목록 페이지 JS 렌더링 차단이 원인이므로 RSS/OpenAPI(한국은행 ECOS·보도자료 RSS) 또는 정책브리핑(korea.kr) 정형 소스를 1차로 두고, 실패 시에만 WebSearch 교차확인으로 폴백. nttId를 못 얻으면 지금처럼 "수집 불가"를 표에 명시(현재 처리 방식 자체는 투명해서 양호).
  3. **네트워크 정책 사전 승인.** 무인 실행에서 샌드박스 해제 프롬프트가 뜨면 중단되므로, 필요한 소스 호스트를 사전 allowlist에 등록하거나 MCP 경로로 전환해 대화형 승인 의존을 제거.
  4. **발행 게이트 유지.** `prebuild`에 물린 `check-privacy.js`가 개인 금액 노출을 자동 차단하는 구조는 무인 실행 안전장치로 적절. 다만 검사기는 "맥락어 + 원화 금액이 같은 줄"에 있을 때만 잡으므로, 표 셀 분리 등으로 우회될 수 있어 정독 리뷰 병행을 계속 권장.
