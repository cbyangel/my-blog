# my-blog

마크다운 파일을 읽어 정적 블로그 웹사이트로 변환하는 프로젝트입니다.

## 목표

- `content/` 안의 마크다운 글을 정적 HTML로 빌드해 `dist/`로 출력한다.
- 깔끔하고 읽기 좋은 디자인 — 편안한 타이포그래피와 넉넉한 여백.
- **다크 모드** 지원 (시스템 설정 자동 감지 + 수동 토글).
- **모바일 반응형** — 작은 화면에서도 잘 읽히게.
- **프레임워크 없이** 순수 HTML / CSS / JavaScript로 구현.

## 아키텍처

정적 사이트 생성(SSG) 방식이다. Node.js 빌드 스크립트가 마크다운을 미리 HTML로 변환한다.

```
my-blog/
├── content/            # 블로그 글 (.md, frontmatter 포함)
│   └── *.md
├── src/
│   ├── templates/      # HTML 템플릿 (레이아웃, 글, 목록)
│   ├── styles/         # CSS (전역 스타일, 테마 변수)
│   └── scripts/        # 클라이언트 JS (다크모드 토글 등)
├── build.js            # 빌드 스크립트: content → dist
├── dist/               # 빌드 산출물 (배포 대상, git 무시)
└── package.json
```

### 빌드 흐름
1. `content/`의 모든 `.md` 파일을 읽는다.
2. frontmatter(제목, 날짜, 태그 등)를 파싱한다.
3. 마크다운 본문을 HTML로 변환한다 (마크다운 파서 라이브러리 사용).
4. 템플릿에 끼워 넣어 각 글의 HTML 페이지를 생성한다.
5. 글 목록(인덱스) 페이지를 생성한다.
6. CSS / JS / 정적 에셋을 `dist/`로 복사한다.

## 기술 스택 규칙

- **런타임 프레임워크 없음** — React/Vue/Svelte 등 사용 금지. 브라우저에는 순수 HTML/CSS/JS만 전달된다.
- 빌드 단계에서만 마크다운 파서 라이브러리(예: `marked`) **하나**를 허용한다. 그 외 의존성은 추가 전에 상의할 것.
- 브라우저로 나가는 코드에는 CDN·외부 스크립트 의존을 넣지 않는다. 필요한 것은 인라인하거나 빌드 시 번들한다.
- 빌드 산출물(`dist/`)은 정적 파일이므로 어떤 정적 호스팅에도 올릴 수 있어야 한다.

## 마크다운 글 형식

각 글은 frontmatter로 시작한다:

```markdown
---
title: 글 제목
date: 2026-07-09
tags: [claude, 개발]
description: 목록과 메타 태그에 쓰일 한 줄 요약
---

본문은 여기서부터...
```

## 디자인 원칙

- **테마**: CSS 커스텀 프로퍼티(`--color-*`)로 라이트/다크 색상을 정의한다.
  - `prefers-color-scheme`로 시스템 설정을 기본값으로 따르고,
  - 토글 버튼으로 수동 전환 + 선택을 `localStorage`에 저장한다.
- **본문**: 최대 폭(약 65~75자)을 제한해 가독성 확보, `line-height` 1.6 이상.
- **반응형**: 상대 단위(rem, %, vw) 사용. 이미지·코드블록·표는 넘칠 때 가로 스크롤 처리.
- **코드블록**: 다크/라이트 모두에서 대비가 충분하도록 스타일링.
- 접근성: 시맨틱 HTML, 충분한 색 대비, 포커스 표시 유지.

## 명령어

> 아직 확정 전 — 구현 시 `package.json`의 scripts에 맞춰 갱신할 것.

```bash
npm run build     # content → dist 빌드
npm run dev       # 로컬 미리보기 서버 (예: 정적 서버 + 감시)
```

## 작업 규칙

- 새 글은 `content/`에 `.md`로 추가하면 빌드 시 자동으로 페이지·목록에 반영되어야 한다.
- 스타일 변경은 다크/라이트, 데스크톱/모바일 네 조합에서 모두 확인한다.
- 변경 후에는 실제로 빌드를 돌려 `dist/`가 정상 생성되는지 확인한다.
