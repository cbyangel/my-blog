# 2048 게임 구현 계획 (spec.md)

## 1. 개요

방향키(↑↓←→)와 터치 스와이프로 4×4 그리드의 숫자 타일을 밀어 같은 숫자끼리 합치는 2048 퍼즐 게임을 만든다. 타일이 합쳐질 때마다 점수가 오르고, 2048 타일을 만들면 승리, 더 이상 움직일 수 없으면 게임 오버다. 현재 점수와 최고 점수(localStorage에 영구 저장)를 함께 보여주며, 언제든 New Game 버튼으로 다시 시작할 수 있다.

이 웹앱은 `/apps/2048/` 폴더 안에 자체 완결(self-contained)된다. 블로그의 다른 파일을 건드리지 않으며, 순수 HTML/CSS/JavaScript만 사용한다. 런타임 프레임워크와 외부 라이브러리는 사용하지 않는다(의존성 0). CDN·외부 스크립트도 없이 데스크톱과 모바일 모두에서 동작하도록 설계한다.

## 2. 파일 구조

```
/apps/2048/
├── index.html   # 게임 마크업 (헤더/점수판/그리드 컨테이너/버튼/메시지 오버레이)
├── style.css    # 레이아웃, 타일 색상 스킴, 반응형, 애니메이션
└── game.js      # 게임 로직 (그리드 상태, 이동/병합, 점수, 판정, 입력 처리, 렌더링)
```

- **index.html**: 시맨틱 마크업으로 페이지 골격을 정의한다. 제목, 점수판(현재/최고), New Game 버튼, 4×4 그리드를 담을 컨테이너, 승리/게임오버 메시지 오버레이를 포함한다. `<link>`로 style.css, `<script defer>`로 game.js를 로드한다.
- **style.css**: CSS 커스텀 프로퍼티로 타일 값별 색상을 정의하고, 그리드/점수판 레이아웃, 반응형(`clamp()`/`vw`/미디어쿼리), 타일 등장·병합 애니메이션을 담는다.
- **game.js**: 프레임워크 없이 순수 JS로 전체 게임 상태와 동작을 담당한다. 모듈 스코프(IIFE 또는 단순 스크립트)로 전역 오염을 피한다.

세 파일 외 추가 파일 없음. 이미지/폰트 등 외부 에셋도 사용하지 않는다(숫자는 텍스트, 아이콘은 유니코드/CSS로 처리).

## 3. 게임 로직 설계

### 3.1 그리드 표현
- 상태는 `grid`: 길이 16의 1차원 배열 또는 4×4 2차원 배열(`grid[row][col]`)로 표현한다. **2차원 배열 채택** — 행/열 접근이 직관적이라 이동 로직 작성이 쉽다.
- 빈 칸은 `0`, 타일은 값(`2, 4, 8, ...`)을 저장한다.
- 전역/모듈 상태: `grid`, `score`(현재 점수), `bestScore`(최고 점수), `won`(승리 후 계속 여부 플래그), `over`(게임오버 플래그).

### 3.2 초기화
- `grid`를 모두 0으로 채운다.
- `spawnTile()`를 2번 호출해 무작위 빈 칸에 타일 2개 배치. 각 타일 값은 90% 확률로 `2`, 10% 확률로 `4`.
- `score = 0`, `won = false`, `over = false`로 리셋 후 렌더링.

### 3.3 이동/병합 알고리즘
핵심은 "한 줄(row/column)을 왼쪽으로 미는" 단일 함수로 모든 방향을 처리하는 것이다.

- `slideRow(row)` — 한 배열을 **왼쪽 기준**으로 처리:
  1. 0을 제거해 값만 남긴다(compact).
  2. 앞에서부터 인접한 두 값이 같으면 하나로 병합(합=값×2), 병합된 값만큼 `score`에 가산. 병합 후 다음 인덱스로 건너뛰어 **한 이동에서 같은 타일이 두 번 병합되지 않도록** 한다.
  3. 다시 0으로 패딩해 길이 4로 맞춘다.
  4. 결과 배열과 획득 점수를 반환.
- 방향별 처리: 모든 방향을 "왼쪽 밀기"로 **정규화**한다.
  - **Left**: 각 행 그대로 `slideRow`.
  - **Right**: 각 행을 뒤집어 `slideRow` 후 다시 뒤집기.
  - **Up**: 열을 추출→행처럼 `slideRow`→열에 되쓰기(전치 이용).
  - **Down**: 열을 뒤집어 `slideRow` 후 되쓰기.
  - 구현상 `transpose()`와 `reverseRows()` 헬퍼로 그리드를 변환→`slideLeft()` 적용→역변환하는 방식으로 통일하면 코드 중복이 줄어든다.
- `moved` 판정: 이동 전 그리드 스냅샷과 이동 후를 비교(직렬화 비교 또는 셀별 비교)해 **실제로 바뀌었는지** 확인.

### 3.4 새 타일 생성
- `moved === true`일 때만 `spawnTile()` 1회 호출.
- 빈 칸 좌표 목록을 모아 무작위로 하나 선택, 90%/10%로 `2`/`4` 배치.

### 3.5 점수 계산
- 병합이 일어날 때마다 병합 결과값(예: 2+2=4면 +4)을 `score`에 더한다.
- 이동 후 `score > bestScore`면 `bestScore` 갱신 및 `localStorage`에 저장.

### 3.6 승리/게임오버 판정
- **승리**: 병합으로 `2048` 타일이 생기면 `won = true` 처리하고 승리 메시지 표시. "계속하기(Keep going)" 옵션을 제공해, 확인 후에는 다시 승리 메시지를 띄우지 않고 게임을 이어갈 수 있게 한다.
- **게임오버**: 이동 후 빈 칸이 하나도 없고, **어떤 방향으로도 병합 가능한 인접 쌍이 없을 때** `over = true`. 판정은 `canMove()`: (1) 빈 칸 존재 여부, (2) 좌우/상하 인접 셀 중 값이 같은 쌍 존재 여부를 검사.

### 3.7 상태 저장
- `bestScore`만 `localStorage`("bestScore-2048" 키)에 저장/복원한다.
- (선택) 진행 중 그리드도 저장해 새로고침 시 복원할 수 있으나, 요구사항 범위에서는 최고 점수 저장만 필수로 한다. 확장 항목으로 표기.

## 4. UI/UX 설계

### 4.1 레이아웃
- 세로 중앙 정렬된 단일 컬럼: **헤더(제목 + 설명)** → **상단 바(점수판 + New Game 버튼)** → **4×4 그리드** → (오버레이) 메시지.
- 그리드는 배경 판(라운드 사각형) 위에 4×4 셀 배경을 깔고, 그 위에 값이 있는 타일을 얹는다. CSS Grid로 4열 레이아웃 구성.

### 4.2 타일 색상 스킴 (값별)
- 저값은 밝은 베이지 계열, 고값으로 갈수록 진한 주황/골드 계열로 대비를 높인다(2048 클래식 팔레트 참고). 예:
  - 2: 연베이지 / 4: 베이지 / 8~64: 주황 계열(점점 진하게) / 128~2048: 골드·앰버 계열.
  - 2·4는 어두운 글자색, 8 이상은 흰 글자색으로 **충분한 색 대비** 확보.
  - 큰 값(3자리·4자리)일수록 `font-size`를 줄여 셀 안에 맞춘다.
- 색상은 CSS 커스텀 프로퍼티 또는 `data-value` 속성 선택자(`.tile[data-value="8"]`)로 매핑.

### 4.3 점수판 배치
- 상단 바 우측(또는 헤더 아래)에 "SCORE"와 "BEST" 두 박스를 나란히 배치. 각 박스는 라벨 + 숫자.
- 병합 시 획득 점수를 `+N` 형태로 잠깐 띄우는 애니메이션은 선택 사항(확장 항목).

### 4.4 반응형 / 터치 처리
- 그리드 크기는 `min()`/`clamp()`/`vw`로 지정해 작은 화면에서 화면 폭에 맞게 축소. 예: `width: min(90vw, 460px)`.
- 폰트·간격도 상대 단위 사용. 좁은 화면에서 점수판과 버튼이 세로로 재배치되도록 미디어쿼리 처리.
- **키보드**: `keydown`에서 Arrow/ WASD 키를 방향으로 매핑, `preventDefault()`로 페이지 스크롤 방지.
- **터치 스와이프**: 그리드 컨테이너에 `touchstart`/`touchend`(또는 `touchmove`)로 시작·끝 좌표 기록 → dx/dy 계산 → 임계값(예: 24px) 이상이면 |dx|,|dy| 비교로 수평/수직 판정, 부호로 방향 결정. `touchmove`에서 스크롤 억제(`preventDefault`, passive:false 고려).
- 접근성: 버튼/오버레이에 시맨틱 요소와 포커스 표시 유지, 색뿐 아니라 숫자로 정보 전달.

## 5. 작업 항목 체크리스트 (Build 순서)

1. `index.html` 골격 작성 — 헤더, 점수판(SCORE/BEST), New Game 버튼, 그리드 컨테이너, 메시지 오버레이, CSS/JS 링크.
2. `style.css` 기본 레이아웃 — 중앙 정렬, 그리드 배경 판 + 4×4 셀, 상대 단위 기반 크기.
3. `style.css` 타일 색상 스킴(값별) + 글자색/폰트 크기 대비 처리.
4. `game.js` 상태 초기화 — `grid`, `score`, `bestScore`(localStorage 로드), 플래그.
5. `spawnTile()` 구현 — 빈 칸 탐색 + 90/10 확률 2/4 배치.
6. `render()` 구현 — 상태를 DOM 타일로 반영, 점수판 갱신.
7. `slideLeft()` + `transpose()`/`reverse` 헬퍼로 4방향 이동·병합 구현, 병합 시 점수 가산.
8. `moved` 판정 후 이동이 있을 때만 `spawnTile()` + 재렌더.
9. 승리(2048) 판정 + 승리 메시지 & "계속하기" 처리.
10. 게임오버 판정(`canMove()`) + 게임오버 메시지.
11. 키보드 입력 핸들러(Arrow/WASD, preventDefault) 연결.
12. 터치 스와이프 핸들러(touchstart/end, 임계값·방향 판정, 스크롤 억제) 연결.
13. New Game 버튼 → 초기화 재실행 연결.
14. `bestScore` localStorage 저장/복원 검증.
15. 반응형 미디어쿼리 마무리 + 데스크톱/모바일, 색 대비 최종 점검.
16. (확장, 선택) 타일 등장·병합 애니메이션, `+N` 점수 팝업, 진행 상태 저장/복원.
```
