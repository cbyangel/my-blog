# Plan 서브에이전트 지침 — 2048 게임

너는 이 블로그 프로젝트에 추가할 웹앱 **2048 게임**의 구현 계획을 작성하는 Plan 담당이다.
**구현은 하지 않는다.** 오직 계획 문서만 작성한다.

## 만들 것
방향키(↑↓←→)로 4×4 그리드의 숫자 타일을 밀어 같은 숫자를 합치는 2048 퍼즐 게임.
점수판 포함.

## 준수할 프로젝트 규칙 (CLAUDE.md)
- 웹앱은 `/apps/2048/` 폴더 안에 자체 완결되어야 한다. 블로그의 다른 파일을 건드리지 않는다.
- 순수 HTML / CSS / JavaScript. 런타임 프레임워크(React/Vue 등) 금지.
- 외부 라이브러리 최소화. (2048은 외부 라이브러리 없이 충분히 구현 가능하니 의존성 0을 목표로 한다.)
- 모바일에서도 사용 가능해야 한다 → 키보드 방향키 + 터치 스와이프 모두 지원.
- 접근성/가독성: 충분한 색 대비, 반응형.

## 게임 기능 요구사항 (계획에 반영할 것)
- 4×4 그리드, 시작 시 무작위 위치에 타일 2개(값 2 또는 4) 생성.
- 방향키 입력 시 해당 방향으로 모든 타일 이동 + 같은 값 인접 타일 병합(한 번의 이동에서 한 타일은 한 번만 병합).
- 이동/병합이 실제로 일어났을 때만 새 타일 1개 생성.
- 점수판: 병합될 때마다 합쳐진 값만큼 점수 증가. 현재 점수 + 최고 점수(localStorage 저장) 표시.
- 승리(2048 타일 도달) 및 게임 오버(움직일 수 없음) 판정과 메시지.
- 다시 시작(New Game) 버튼.
- 모바일 터치 스와이프 지원.

## 작성할 산출물
`/apps/2048/spec.md` 파일 하나. 다음을 포함한다:
1. **개요** — 무엇을 만드는지 한두 문단.
2. **파일 구조** — `/apps/2048/` 안에 만들 파일 목록과 각 파일 역할 (예: index.html, style.css, game.js).
3. **게임 로직 설계** — 그리드 표현 방식, 이동/병합 알고리즘, 새 타일 생성, 점수 계산, 승리/게임오버 판정, 상태 저장.
4. **UI/UX 설계** — 레이아웃, 타일 색상 스킴(값별), 점수판 배치, 반응형/터치 처리.
5. **작업 항목 체크리스트** — Build 단계에서 구현할 항목들을 순서대로.

## 주의
- 실제 index.html/style.css/game.js 코드를 작성하지 마라. 오직 `spec.md`만 작성한다.
- 최종 응답에는 작성한 `spec.md`의 핵심 요약(파일 구조 + 주요 결정)을 담아라. 이 응답이 곧 반환값이다.
