## 만들 파일
1. `index.html` — 헤더, 점수판(SCORE / BEST), New Game 버튼, 4×4 그리드 컨테이너, 승리/게임오버 메시지 오버레이. CSS와 JS를 각각 `style.css`, `game.js`로 링크.
2. `style.css` — 레이아웃, 값별 타일 색상(클래식 2048 팔레트), 반응형(clamp/vw로 모바일 대응), 부드러운 등장/이동 느낌의 트랜지션.
3. `game.js` — 게임 전체 로직.

## 기술 규칙
- 순수 HTML/CSS/JS. 프레임워크·외부 라이브러리·CDN·외부 이미지 **금지**. 의존성 0.
- 페이지를 파일로 직접 열어도(`file://`) 동작해야 한다.

## 기능 요구사항 (spec 기준, 반드시 충족)
- 4×4 그리드, 시작 시 무작위 타일 2개(2 또는 4).
- 방향키(↑↓←→) 및 WASD로 이동. 페이지 스크롤 방지(preventDefault).
- 모바일 터치 스와이프 지원(가로/세로 판정, 임계값 ~24px, 스크롤 억제).
- 이동/병합이 실제 발생했을 때만 새 타일 1개 생성(90% 2, 10% 4).
- 한 번의 이동에서 한 타일은 한 번만 병합. 병합값만큼 점수 증가.
- 점수판: 현재 점수 + 최고 점수. 최고 점수는 localStorage("bestScore-2048")에 저장.
- 2048 도달 시 승리 메시지(계속하기 가능), 이동 불가 시 게임오버 메시지.
- New Game 버튼으로 초기화.
- 접근성/가독성: 충분한 색 대비, 포커스 표시 유지, 큰 값 타일은 폰트 축소.
