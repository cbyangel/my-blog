## 검증 항목
1. **정합성** — spec.md의 파일 구조/기능 요구사항이 실제 코드에 반영됐는가.
2. **게임 로직 정확성** — 이동/병합(한 이동당 1회 병합), 이동 발생 시에만 타일 생성, 점수 계산, 승리(2048)·게임오버(canMove) 판정. 가능하면 `game.js`의 핵심 로직을 node로 실제 실행해 여러 케이스를 검증하라(임시 테스트 파일을 apps/2048/ 안에 만들어 실행하고, 끝나면 반드시 삭제).
3. **정적 오류** — `node --check game.js`, HTML 구조 유효성, JS와 HTML의 id/클래스 참조 일치, CSS 선택자와 마크업 일치.
4. **기술 규칙 준수** — 순수 HTML/CSS/JS, 외부 라이브러리·CDN·외부 이미지 없음(의존성 0), file://로 열어도 동작.
5. **입력** — 키보드(방향키/WASD, preventDefault로 스크롤 방지) + 터치 스와이프 처리 존재/정확성.
6. **저장** — 최고점수 localStorage 저장/복원, file:// 예외 처리.
7. **접근성/반응형** — 색 대비, 포커스 표시, 모바일 반응형.
8. 가능하면 브라우저(헤드리스 포함)로 실제 렌더링/동작을 확인하라. 브라우저 구동이 불가하면 그 사실을 review.md에 명시하고 정적·로직 검증으로 대체하라.

## 문제 발견 시
- 발견한 문제는 심각도(치명/중간/경미)와 함께 기록한다.
- **apps/2048/ 폴더 안에서만** 직접 수정해도 된다(블로그의 다른 파일은 절대 건드리지 마라). 수정했다면 무엇을 어떻게 고쳤는지 review.md에 남긴다.
- 수정 후 재검증한다.
